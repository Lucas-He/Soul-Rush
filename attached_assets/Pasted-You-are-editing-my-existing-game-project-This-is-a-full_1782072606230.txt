ANTI-LAG / PERFORMANCE FIX PASS
===============================

Mandatory:
The fight must look premium and run smoothly.

Performance target:

* smooth and responsive gameplay
* no obvious frame drops during attacks
* no stutter during transitions
* no heavy lag from boss rendering, arena rendering, or VFX

Required optimizations:

1. object pooling for:

* projectiles
* warning markers
* dash echoes
* lane indicators
* cage segments
* impact effects
* shard bursts
* spiral hazards if reused

2. reuse nodes/materials:

* do not spawn too many temporary nodes every attack
* reuse sprites, meshes, particles, line renderers, shaders
* avoid duplicate UI creation
* avoid repeated allocations inside attack loops

3. limit particles:

* use fewer stronger particles
* short lifetimes
* hard cap emission counts
* remove decorative emitters that do not aid readability

4. limit transparency stacking:

* fewer overlapping glow layers
* no large lingering transparent panels
* afterimages few and short-lived
* no blur+bloom+glow+particles stacking excessively

5. simplify collision:

* clean rectangles, lanes, lines, arcs, wedges
* no overly complex collision for purely visual FX
* hitboxes simple and honest

6. reduce full-screen effects:

* no constant distortion
* no heavy blur
* restrained bloom
* minimal post-process usage

7. avoid wasteful per-frame logic:

* cache pattern data
* precompute telegraph geometry where possible
* no unnecessary string/UI updates every frame

8. clean up fast:

* return pooled objects immediately
* remove old warnings / glows / trails quickly
* clear segment effects cleanly
